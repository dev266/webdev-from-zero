Concepts

Introduction

Before writing a single line of code, let's understand what a website actually is and how the internet delivers it to your screen. This foundation will make everything else click.

What is a Website?

A website is a collection of files stored on a computer somewhere in the world (called a server). When you visit a website, your computer (the client) asks that server to send those files, and your browser turns them into the page you see.

Every website is made of three types of files:

File TypeExtensionPurpose
HTML.htmlThe structure — headings, paragraphs, buttons, forms
CSS.cssThe style — colors, fonts, layout, spacing
JavaScript.jsThe behavior — clicks, animations, talking to servers

Analogy: HTML is the skeleton, CSS is the skin and clothes, JavaScript is the muscles that make it move.

How the Internet Works

The internet is a global network of computers that can talk to each other using agreed-upon rules (called protocols). The most important one for websites is HTTP (HyperText Transfer Protocol).

Your Browser
Client
── HTTP Request ──▶
Web Server
Server
◀── HTTP Response ──
Your Browser
Gets the files

The server sends back HTML, CSS, and JavaScript files. Your browser reads them and draws the page.

What Happens When You Type a URL?

  1. You type https://example.com and press Enter.
  2. Your browser asks a DNS server: "What is the IP address of example.com?" — like looking up a phone number in a directory.
  3. The DNS server replies with an IP address like 93.184.216.34.
  4. Your browser connects to that IP address and sends an HTTP GET request asking for the homepage.
  5. The server reads the request, finds the right files, and sends them back.
  6. Your browser receives HTML, reads it, then fetches any linked CSS and JS files.
  7. Your browser renders (draws) the page from all those files.

You just learned what most people who "use the internet" never understand. This is how every website on Earth works — from Google to your local bakery's homepage.

Frontend

Time to write real code. You'll create three files: index.html, styles.css, and script.js. By the end of this module you'll have a working web page that looks good and responds to clicks.

Step 1 — Create Your Project Folder

  1. Create a new folder on your Desktop. Name it mywebsite.
  2. Open VS Code. Click File → Open Folder and select that folder.
  3. You now have an empty project. All three files go in this same folder.

Step 2 — Create index.html

This is your main page. Every website starts with a file named exactly index.html — the browser loads this file first automatically.

index.html
HTML
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Website</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>

  <header>
    <h1>Hello, World!</h1>
    <p>My first ever website.</p>
  </header>

  <main>
    <h2>About Me</h2>
    <p>I am learning to build websites from scratch!</p>

    <button id="myBtn">Click Me</button>
    <p id="message"></p>
  </main>

  <footer>
    <p>Built by me 💪</p>
  </footer>

  <!-- JavaScript goes at the bottom, before </body> -->
  <script src="script.js"></script>
</body>
</html>

Why index.html? Web servers are configured to look for a file named index.html when someone visits a folder. It's a universal convention.

Step 4 — Create script.js

Create a third file named exactly script.js. This adds interactive behavior.

script.js
JavaScript
// Wait until the page is fully loaded

  // Find the button element by its id
  const btn = document.getElementById('myBtn');

  // Find the message paragraph
  const message = document.getElementById('message');

  // When the button is clicked, run this function
  btn.addEventListener('click', function() {
    message.textContent = 'You clicked the button! JavaScript is working!';
  });

});

  3. Click the button — you'll see a green message appear. That's JavaScript in action!

Backend (Node.js)

The frontend lives in the browser. The backend lives on a server. You'll build a backend using Node.js and Express — a popular, beginner-friendly combination.

What is a Server?

A server is a program that listens for requests and responds to them. When your frontend (running in the browser) needs data — say, a list of users — it sends a request to your backend server. The server fetches the data and sends it back.

Node.js lets you run JavaScript on the server — the same language you already know from the frontend. Express is a library that makes building servers much simpler.

Step 1 — Install Node.js

  1. Go to nodejs.org and download the LTS version (Long-Term Support — the stable one).
  2. Run the installer and click through the defaults.
  3. Open a terminal (in VS Code: Terminal → New Terminal) and run:
Terminal
BASH
node --version
npm --version

If you see version numbers (like v20.11.0), Node.js is installed correctly.

Step 2 — Create package.json

Every Node.js project needs a package.json file. It's a configuration file that tracks your project name, version, and all the libraries (packages) your project uses.

In your project folder, run:

Terminal
BASH
npm init -y

This creates package.json automatically. The -y flag says "yes" to all default options.

Step 3 — Install Express

Terminal
BASH
npm install express

This downloads Express and saves it in a folder called node_modules. It also updates package.json with "express" in the dependencies list.

Step 4 — Create server.js

Create a new file named exactly server.js in your project folder.

server.js
JavaScript
// Import the Express library
const express = require('express');

// Create an Express app
const app = express();

// Define what PORT the server will listen on
const PORT = 3000;

// Allow the server to read JSON from requests
app.use(express.json());

// Serve your HTML/CSS/JS files from the same folder
app.use(express.static('.'));

// Define a GET route at /api/hello
app.get('/api/hello', (req, res) => {
  res.json({ message: 'Hello from the server!' });
});

// Start the server
app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});

Step 5 — Run the Server

Terminal
BASH
node server.js

You should see: Server running at http://localhost:3000

Open your browser and visit http://localhost:3000/api/hello. You'll see:

Browser Response
JSON
{ "message": "Hello from the server!" }

You just built a real API endpoint! /api/hello is a route. When someone visits that URL, the server runs your function and sends back JSON data.

Key Concepts

TermMeaning
require()Import a library or file (like Python's import)
express()Creates your app object — everything hangs off this
app.get()Define a route that responds to GET requests
reqThe incoming request (contains URL params, body, etc.)
resThe response you send back
res.json()Send JSON data back to the client
app.listen(PORT)Start the server on a specific port

Connect Frontend ↔ Backend

Your frontend (in the browser) and backend (on the server) need to talk to each other. They do this via HTTP requests. In JavaScript, you use the fetch() function to send requests from the browser to your server.

How HTTP Requests Work

MethodUsed forExample
GETRead / fetch dataGet list of users
POSTCreate / send dataSubmit a form
PUTUpdate existing dataEdit a user profile
DELETERemove dataDelete a post

Step 1 — Add a Button to index.html

Update your index.html — add a button to call the backend:

index.html — inside <main>
HTML
<button id="fetchBtn">Ask the Server</button>
<p id="serverResponse"></p>

Step 2 — Use fetch() in script.js

Update script.js to call the backend when the button is clicked:

script.js
JavaScript
document.getElementById('fetchBtn').addEventListener('click', async function() {

  // Send a GET request to our backend
  const response = await fetch('/api/hello');

  // Parse the JSON response
  const data = await response.json();

  // Display the message from the server
  document.getElementById('serverResponse').textContent = data.message;

});

Step 3 — Add a POST Route to server.js

POST is used when the frontend sends data TO the server (like a form). Add this route to server.js:

server.js — add this route
JavaScript
// POST route — receives data from the frontend
app.post('/api/greet', (req, res) => {
  const { name } = req.body; // Get "name" from the request body
  res.json({ message: `Hello, ${name}! The server says hi.` });
});

Step 4 — Call the POST Route from the Frontend

script.js — POST example
JavaScript
async function greetUser(name) {
  const response = await fetch('/api/greet', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ name: name })
  });

  const data = await response.json();
  console.log(data.message);
}

greetUser('Alice'); // → "Hello, Alice! The server says hi."

Database (MySQL)

A database stores your data permanently. Without a database, all your data disappears when the server restarts. MySQL is one of the most popular databases in the world — it stores data in tables, like spreadsheets.

What is a Database?

Think of a database as a collection of tables. Each table is like a spreadsheet:

idnameemail
1Alicealice@example.com
2Bobbob@example.com
3Carolcarol@example.com

This is a users table. Each row is one user. Each column is one piece of data. The id column auto-increments — MySQL assigns it automatically.

Installing MySQL

  1. Go to dev.mysql.com/downloads/mysql and download MySQL Community Server.
  2. Run the installer. When asked to set a root password, write it down — you'll need it later.
  3. After installation, open MySQL Workbench (installed alongside MySQL) or use the terminal.

Remember your MySQL root password. You'll use it to connect from Node.js. If you forget it, resetting it is painful.

Creating Your Database

Open MySQL Workbench (or the MySQL terminal) and run these SQL commands:

MySQL — setup.sql
SQL
-- Create a new database
CREATE DATABASE myapp;

-- Switch to using that database
USE myapp;

-- Create a users table
CREATE TABLE users (
  id    INT AUTO_INCREMENT PRIMARY KEY,
  name  VARCHAR(100) NOT NULL,
  email VARCHAR(100) NOT NULL UNIQUE
);

-- Insert a test row
INSERT INTO users (name, email)
VALUES ('Alice', 'alice@example.com');

-- Read all rows
SELECT * FROM users;

Essential SQL Commands

SQL CommandWhat it does
CREATE DATABASE nameCreate a new database
USE nameSelect which database to work with
CREATE TABLECreate a new table with defined columns
INSERT INTOAdd a new row to a table
SELECT * FROMRead all rows from a table
WHERE id = 1Filter to only matching rows
UPDATE table SETEdit an existing row
DELETE FROMRemove rows

Column Types

TypeUsed forExample

VARCHAR(n)Text up to n charactersname, email, title

BOOLEANTrue or falseis_active, is_verified

Connect Backend ↔ MySQL

Now you'll wire Node.js to MySQL. The backend will insert data into the database and read it back. You'll also learn about .env files — the right way to store secret credentials.

Step 1 — Install mysql2 and dotenv

Terminal
BASH
npm install mysql2 dotenv

mysql2 is the Node.js driver for MySQL. dotenv loads environment variables from a .env file.

Step 2 — Create .env

Create a file named exactly .env (yes, the dot is part of the name). Store your database credentials here.

.env
ENV
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password_here
DB_NAME=myapp
PORT=3000

Never commit .env to Git. Add .env to your .gitignore file. This file contains passwords — if you push it to GitHub, anyone can see your credentials.

Step 3 — Create .gitignore

Create a file named .gitignore to tell Git which files to ignore:

.gitignore
TEXT
node_modules/
.env

Step 4 — Update server.js with MySQL

server.js
JavaScript
require('dotenv').config(); // Load .env variables

const express = require('express');
const mysql   = require('mysql2/promise');

const app = express();
app.use(express.json());
app.use(express.static('.'));

// Create a connection pool (reuses connections efficiently)
const pool = mysql.createPool({
  host:     process.env.DB_HOST,
  user:     process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  waitForConnections: true,
});

// GET all users from the database
app.get('/api/users', async (req, res) => {
  try {
    const [rows] = await pool.query('SELECT * FROM users');
    res.json(rows);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

// POST — add a new user to the database
app.post('/api/users', async (req, res) => {
  const { name, email } = req.body;
  try {
    const [result] = await pool.query(
      'INSERT INTO users (name, email) VALUES (?, ?)',
      [name, email]      // ? placeholders prevent SQL injection
    );
    res.json({ id: result.insertId, name, email });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

app.listen(process.env.PORT, () => {
  console.log(`Server running on port ${process.env.PORT}`);
});

Always use parameterized queries — the ? placeholders. Never concatenate user input directly into SQL strings. That's called SQL injection and it's a serious security vulnerability.

Full Project — User Registration App

Let's build a complete working app: a form that lets users enter their name and email, saves it to MySQL, and displays all registered users — all connected end-to-end.

What You'll Build

  1. A form in index.html where users type their name and email.
  2. A click handler in script.js that sends the data to the backend via fetch().
  3. A POST route in server.js that saves it to MySQL.
  4. A GET route that fetches all users from MySQL and returns them.
  5. The frontend displays the list of users on the page.

index.html — Full Version

index.html
HTML
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>User Registration</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <h1>User Registration</h1>

  <form id="registerForm">
    <input type="text"  id="nameInput"  placeholder="Your name"  required>
    <input type="email" id="emailInput" placeholder="Your email" required>
    <button type="submit">Register</button>
  </form>

  <p id="status"></p>

  <h2>Registered Users</h2>
  <ul id="userList"></ul>

  <script src="script.js"></script>
</body>
</html>

script.js — Full Version

script.js
JavaScript
// Load users when the page opens
document.addEventListener('DOMContentLoaded', loadUsers);

// Handle form submission
document.getElementById('registerForm').addEventListener('submit', async function(e) {
  e.preventDefault(); // Stop form from reloading the page

  const name  = document.getElementById('nameInput').value;
  const email = document.getElementById('emailInput').value;

  const res = await fetch('/api/users', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ name, email })
  });

  const data = await res.json();

  if (data.error) {
    document.getElementById('status').textContent = 'Error: ' + data.error;
  } else {
    document.getElementById('status').textContent = 'Registered successfully!';
    document.getElementById('registerForm').reset();
    loadUsers(); // Refresh the list
  }
});

async function loadUsers() {
  const res   = await fetch('/api/users');
  const users = await res.json();
  const list  = document.getElementById('userList');

  list.innerHTML = users.map(u =>
    `<li><strong>${u.name}</strong> — ${u.email}</li>`
  ).join('');
}

server.js — Full Version

server.js
JavaScript
require('dotenv').config();
const express = require('express');
const mysql   = require('mysql2/promise');

const app  = express();
const pool = mysql.createPool({
  host:     process.env.DB_HOST,
  user:     process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
});

app.use(express.json());
app.use(express.static('.'));

app.get('/api/users', async (req, res) => {
  const [rows] = await pool.query('SELECT * FROM users ORDER BY id DESC');
  res.json(rows);
});

app.post('/api/users', async (req, res) => {
  const { name, email } = req.body;
  if (!name || !email) return res.status(400).json({ error: 'Name and email required' });

  try {
    const [result] = await pool.query(
      'INSERT INTO users (name, email) VALUES (?, ?)',
      [name, email]
    );
    res.json({ id: result.insertId, name, email });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

app.listen(process.env.PORT || 3000, () =>
  console.log('Server running on port 3000')
);

Start the server with node server.js, open http://localhost:3000, fill in the form, and watch users appear in real time — stored in MySQL.

Folder Structure

Knowing what every file is and why it exists prevents confusion. Here is the complete folder structure for your project and a clear explanation of each file.

Your Complete Project

mywebsite/ │ ├── index.html ← Main page (what users see) ├── styles.css ← All visual styling ├── script.js ← Frontend JavaScript (runs in browser) ├── server.js ← Backend server (runs in Node.js) ├── package.json ← Project metadata + dependencies ├── package-lock.json ← Exact versions (auto-generated) ├── .env ← Secret credentials (never commit!) ├── .gitignore ← Tells Git what NOT to track └── node_modules/ ← Installed packages (auto-generated)

File-by-File Breakdown

FileExtensionRuns wherePurpose
index.html.htmlBrowserThe HTML structure of your page — the only file the browser loads first
styles.css.cssBrowserColors, fonts, layout, spacing — makes HTML look good
script.js.jsBrowserFrontend logic — event listeners, fetch() calls, DOM manipulation
server.js.jsNode.js (server)Backend server — handles API routes, talks to MySQL
package.json.json—Lists your project name and all npm dependencies
.env(none)—Secret environment variables — DB password, API keys
.gitignore(none)—List of files Git should never track
node_modules/(folder)—All downloaded npm packages — never edit manually

Important Rules

  • index.html must be named exactly that — web servers look for this file automatically.
  • Never edit node_modules — npm manages it. Delete and run npm install to restore it.
  • Never commit .env — it contains passwords. Add it to .gitignore.
  • package-lock.json should be committed — it locks exact dependency versions for your team.
  • Extensions matter — styles.css not styles.CSS, server.js not server.JS.

When collaborators clone your project, they run npm install to regenerate node_modules from package.json. That's why node_modules is gitignored.

Part 1 — Deploy Frontend to Netlify

Netlify hosts static files (HTML, CSS, JS) for free.

  1. Create a free account at netlify.com.
  2. Push your project to GitHub (see steps below).
  3. In Netlify, click Add New Site → Import from Git.
  4. Connect your GitHub repo.
  5. Set Build command to blank (no build step needed for plain HTML).
  6. Set Publish directory to . (root folder).
  7. Click Deploy. You'll get a live URL like yoursite.netlify.app.

Push to GitHub First

Terminal — git setup
BASH
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/mywebsite.git
git push -u origin main

Part 2 — Deploy Backend to Render

Render hosts Node.js servers for free (with some limits).

  1. Create a free account at render.com.
  2. Click New → Web Service.
  3. Connect your GitHub repo.
  4. Set Build Command to npm install.
  5. Set Start Command to node server.js.
  6. Add your environment variables (DB_HOST, DB_USER, DB_PASSWORD, DB_NAME) in the Environment tab — not by uploading .env.
  7. Click Create Web Service. You'll get a URL like myapp.onrender.com.

Free Render services spin down after 15 minutes of inactivity and take ~30 seconds to wake up on the next request. This is fine for learning — upgrade for production apps.

Part 4 — Update Frontend to Point to Deployed Backend

In your frontend's script.js, the fetch URL needs to point to your deployed backend:

script.js — update for production
JavaScript
// Development: relative URL works because frontend + backend run together
const API = window.location.hostname === 'localhost'
  ? 'http://localhost:3000'
  : 'https://myapp.onrender.com'; // Your Render URL

// Then use the API variable in all fetch calls:
const res = await fetch(`${API}/api/users`);

3 — Node.js and npm

Node.js runs JavaScript outside the browser (on your server). npm (Node Package Manager) comes with it and lets you install third-party libraries.

Common npm commands
BASH
npm install express          # Install a package
npm install                  # Install all packages from package.json
npm install -g nodemon       # Install globally (available everywhere)
nodemon server.js            # Run server, auto-restart on file changes
npm list                     # Show installed packages
npm uninstall express        # Remove a package

Install nodemon globally: npm install -g nodemon. Then use nodemon server.js instead of node server.js. It restarts the server automatically every time you save a file — saves you tons of Ctrl+C.

Common Errors

Every developer hits these errors. Here's exactly what they mean and how to fix them fast.

Server Errors

Cannot find module 'express'
Error: Cannot find module 'express'
Fix: You forgot to install dependencies. Run npm install in your project folder.
EADDRINUSE — Port already in use
Error: listen EADDRINUSE: address already in use :::3000
Fix: Another process is already using port 3000. Either stop it with kill $(lsof -ti:3000) (Mac/Linux) or change your PORT in .env to 3001.
process.env variables are undefined
DB_HOST is undefined
Fix: Make sure require('dotenv').config() is the very first line in server.js, before any other requires. Also check your .env file exists and has no typos.

Database Errors

Access denied for user 'root'
Error: Access denied for user 'root'@'localhost' (using password: YES)
Fix: Wrong password in your .env file. Check DB_PASSWORD matches the password you set when installing MySQL.
Unknown database
Error: Unknown database 'myapp'
Fix: You haven't created the database yet. Open MySQL Workbench and run CREATE DATABASE myapp;
Table doesn't exist
Error: Table 'myapp.users' doesn't exist
Fix: Run the CREATE TABLE SQL from Module 5. Make sure you ran USE myapp; first.

Frontend/CORS Errors

CORS Policy Error
Access to fetch at 'https://myapp.onrender.com/api/users' from origin 'https://mysite.netlify.app' has been blocked by CORS policy
Fix: Add the cors package to your server (see Module 9). Make sure the origin in your cors config exactly matches your Netlify URL (no trailing slash).
fetch() returning HTML instead of JSON
SyntaxError: Unexpected token '<', "<!DOCTYPE..." is not valid JSON
Fix: Your server returned an HTML error page instead of JSON. Open the Network tab in DevTools, click the failed request, and read the actual response to see the real error.
404 Not Found on API route
GET /api/users 404 Not Found
Fix: The route doesn't exist on your server. Check spelling — /api/users vs /api/user. Also make sure your server is actually running.

Deployment Errors

Build fails on Render
npm error: missing script: start
Fix: Add a start script to package.json:"scripts": { "start": "node server.js" }
Environment variables missing in production
Cannot read properties of undefined (reading 'DB_HOST')
Fix: You never commit .env. Add the variables manually in Render's Environment tab. Every deployment platform has an "Environment Variables" section — add them there.

Quick Debug Checklist

  1. Is the server running? Check the terminal for errors.
  2. Is the URL correct? Open it directly in the browser and check the response.
  3. Open DevTools → Console tab. Any red errors?
  4. Open DevTools → Network tab. Click the failing request and read the response body.
  5. Add console.log() statements to narrow down where data is lost.
  6. Check .env — is it in the right folder? Are variable names spelled exactly right?
